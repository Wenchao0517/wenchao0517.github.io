SkillScape: Career Exploration Platform

The Challenge

Traditional career exploration (job descriptions, career fairs) fails to provide accurate understanding of day-to-day professional tasks. Students lack confidence in career decisions due to lack of real-world exposure.

The Solution

Created seamless cross-device platform with:

AR Simulations Career Browsing Skill Assessments Personalized Recommendations

Design Process

Phase Key Activities
Research User interviews, target audience analysis, pain point mapping
Ideation Card-based navigation design, progressive information disclosure
Prototyping High-fidelity Figma prototype with 9+ interactive screens
Testing Usability testing with 2 target users (final-year students)
Refinement Iterated based on user feedback on button visibility and navigation

Design Decisions

Color Palette: Purple/magenta gradient (optimism, creativity, innovation)
Typography: Clean, minimalist interface reducing cognitive load
Layout: Card-based navigation with visual hierarchy

Results & Impact

Satisfaction Score: 4/5
Perceived Difficulty: 1/5 (highly intuitive)
Successful cross-device functionality
AR integration validated for career exploration

Usability Testing Insights

User 1 Issues: Small save button (confusion 3/5) ‚Üí Enlarged button in revision. Assessment results unclear location (confusion 4/5) ‚Üí Moved to dedicated profile section.

User 2 Feedback: Requested tooltips for AR features ‚Üí Added onboarding support. Interface meets expectations ‚Üí Positive validation.

Key Iterations

  1. Enhanced profile section visibility
  2. Enlarged interactive buttons for better discoverability
  3. Added tooltips/onboarding for AR features
  4. Clarified back navigation buttons
  5. Improved assessment results display

My Contribution

Led visual design refinement, coordinated usability testing, contributed to prototype design iterations, documented all design decisions and research findings.

Driver Safety System: AI-Powered Posture & Fatigue Detection

The Challenge

Poor driving posture causes physical strain and long-term health issues. Driver fatigue causes accidents and reduced alertness. Existing systems lack real-time, actionable feedback and clear visibility into system logic.

The Solution

Designed intuitive interface with real-time feedback and safe interaction during driving:

Real-time Monitoring Steering Wheel Control Fatigue Levels Posture Suggestions

Design Process

Phase Key Activities
Research Ergonomic analysis, safety requirements, existing system review
Ideation Paper prototyping, task flow design, feedback mechanisms
Heuristic Eval Expert evaluation by 3 reviewers, severity ratings, issues identified
Iteration Multiple design rounds based on feedback (steering wheel buttons, status visibility)
Testing 3 rounds of usability testing with different user groups

Critical Design Decisions

Steering Wheel Button: Eliminates dangerous screen-touching while driving
Fatigue Display: Real-time indicators (Low/High) build trust in AI logic
Posture Labels: Specific feedback ("Slouching", "Leaning") guides correction
Dual Feedback: Audio + visual alerts reach drivers in different contexts

Results & Impact

Safety-first UI eliminates distraction-prone interactions
System transparency increased user trust
3 successful testing iterations with validation
Specific feedback enables faster corrective action

Heuristic Evaluation Findings

Issue Severity Solution
Screen touch unsafe while driving 4/5 Added steering wheel button control
System status unclear 3/5 Added info prompts explaining sensor activation
Fatigue threshold abrupt 3-4/5 Added real-time fatigue level display with visual indicators

Usability Testing - 3 Rounds

Round 1: User confused about specific posture issue ‚Üí Added "Slouching", "Leaning" labels

Round 2: Eye icon fatigue indicator too small ‚Üí Increased icon size and contrast

Round 3: No confirmation when posture corrected ‚Üí Added status confirmation feedback

Key Learning

In safety-critical design, transparency about system logic (what the AI is detecting) builds trust more than perfect functionality. Users need to understand WHY the system alerts them.

My Contribution

Coordinated usability testing, documented heuristic evaluation results, led design iterations based on safety feedback, created before/after design comparisons.

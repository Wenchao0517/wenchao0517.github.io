Driver Safety System

AI-Powered Posture & Fatigue Detection

Overview: An intelligent in-vehicle system that monitors driver posture and detects fatigue in real-time, promoting driver safety and long-term ergonomic health.

üéØ Problem

Poor driving posture causes physical strain and long-term health issues. Driver fatigue causes accidents and reduced alertness. Existing systems lack real-time, actionable feedback and clear visibility into system logic.

üí° Solution

Designed intuitive interface with real-time feedback and safe interaction during driving:

Real-time Monitoring Steering Wheel Control Fatigue Levels Posture Suggestions

üìä Design Process

Phase Key Activities
Research Ergonomic analysis, safety requirements, existing system review
Ideation Paper prototyping, task flow design, feedback mechanisms
Heuristic Eval Expert evaluation by 3 reviewers, severity ratings, issues identified
Iteration Multiple design rounds based on feedback (steering wheel buttons, status visibility)
Testing 3 rounds of usability testing with different user groups

üé® Critical Design Decisions

Steering Wheel Button: Eliminates dangerous screen-touching while driving
Fatigue Display: Real-time indicators (Low/High) build trust in AI logic
Posture Labels: Specific feedback ("Slouching", "Leaning") guides correction
Dual Feedback: Audio + visual alerts reach drivers in different contexts

üìà Results

‚úì Safety-first UI eliminates distraction-prone interactions
‚úì System transparency increased user trust
‚úì 3 successful testing iterations with validation
‚úì Specific feedback enables faster corrective action

üß™ Heuristic Evaluation Findings

Issue Severity Solution
Screen touch unsafe while driving 4/5 Added steering wheel button control
System status unclear ("artificial") 3/5 Added info prompts on startup explaining sensor activation
Fatigue threshold abrupt 3-4/5 Added real-time fatigue level display with visual indicators

üîÑ Usability Testing - 3 Rounds

Round 1: User confused about specific posture issue ‚Üí Added "Slouching", "Leaning" labels

Round 2: Eye icon fatigue indicator too small ‚Üí Increased icon size and contrast

Round 3: No confirmation when posture corrected ‚Üí Added status confirmation feedback

üéØ Key Learning

In safety-critical design, transparency about system logic (what the AI is detecting) builds trust more than perfect functionality. Users need to understand WHY the system alerts them.

üë• My Specific Contribution

Coordinated usability testing, documented heuristic evaluation results, led design iterations based on safety feedback, created before/after design comparisons.
